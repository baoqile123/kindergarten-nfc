# 🏫 幼儿园NFC签到系统迁移操作流程

## 📋 文档概述

本文档提供了从freekit.dev迁移到GitHub Pages的**完整操作流程**，确保幼儿园户外运动签到系统的稳定运行。

### 迁移原因
- ✅ 解决部分地区访问freekit.dev不稳定的问题
- ✅ 提供长期免费的全球CDN托管
- ✅ 支持自定义域名扩展
- ✅ 保持所有功能不变

### 预计时间
- **技术部署**：30分钟
- **手环更新**：8.5小时（分批进行）
- **教师培训**：1小时

## 🎯 第一阶段：技术部署（今日完成）

### 步骤1：创建GitHub账号（如无）
1. 访问 https://github.com/signup
2. 填写：用户名、邮箱、密码
3. 验证邮箱地址
4. 登录GitHub

### 步骤2：创建GitHub仓库
1. 访问 https://github.com/new
2. 填写信息：
   - **Repository name**：`kindergarten-nfc`（建议）
   - **Description**：幼儿园户外运动NFC签到系统
   - **Public**：选择公开仓库
   - **不要勾选**："Add a README file"
3. 点击"Create repository"

### 步骤3：上传部署文件
#### 方法A：Web界面上传（推荐）
1. 进入新建的仓库：`https://github.com/[用户名]/kindergarten-nfc`
2. 点击"Add file" → "Upload files"
3. 打开本地的`github_pages_deploy`文件夹
4. **全选所有文件**，拖拽到上传区域
5. 点击"Commit changes"

#### 方法B：使用Git命令（可选）
```bash
# 1. 克隆仓库
git clone https://github.com/[用户名]/kindergarten-nfc.git

# 2. 进入仓库目录
cd kindergarten-nfc

# 3. 复制所有文件（从github_pages_deploy文件夹）
cp -r ../github_pages_deploy/* .

# 4. 提交更改
git add .
git commit -m "部署NFC签到系统 v1.0"
git push
```

### 步骤4：启用GitHub Pages
1. 进入仓库：`https://github.com/[用户名]/kindergarten-nfc`
2. 点击顶部"Settings"选项卡
3. 左侧菜单选择"Pages"
4. 配置页面：
   - **Source**：选择"Deploy from a branch"
   - **Branch**：选择"main"，路径`/(root)`
5. 点击"Save"

### 步骤5：等待部署完成
- 等待约1-3分钟
- 刷新页面查看状态（显示绿色✅即成功）
- 访问地址将显示：`https://[用户名].github.io/kindergarten-nfc/`

## 📱 第三阶段：手环更新（本周内分批完成）

### 手环链接格式
```
https://[用户名].github.io/kindergarten-nfc/nfc-checkin.html#姓名
```

### 更新工具：NFC Tools APP
1. **安装APP**：在手机应用商店搜索"NFC Tools"安装
2. **打开APP**：选择"写"选项卡
3. **选择URL类型**：点击"添加记录" → "URL/URI"
4. **输入链接**：`https://[用户名].github.io/kindergarten-nfc/nfc-checkin.html#姓名`
5. **写入手环**：将手环靠近手机NFC区域
6. **验证成功**：听到提示音，APP显示"写入成功"

### 批量更新策略
#### 方案A：按班级分批（推荐）
```
第1天：中一班（29人） → 约2.5小时
第2天：大一班（17人） → 约1.5小时
第3天：小一班（28人） → 约2.5小时
第4天：小二班（27人） → 约2小时
```

#### 方案B：按区域分组
- 每天更新一个运动区域的相关手环
- 便于现场测试验证

#### 方案C：全员集中
- 选择半天时间集中更新
- 需要多名老师协助
- 效率最高，但组织复杂

## 🔄 第六阶段：并行运行与切换（1周）

### 并行运行策略
#### 第1-3天：双系统并行
- **旧系统**：freekit.dev链接继续使用
- **新系统**：GitHub Pages链接开始使用
- **数据对比**：对比两个系统的签到记录

#### 第4-5天：逐步切换
- **区域切换**：每天切换2个区域到新系统
- **手环混合**：新旧手环同时可用
- **问题收集**：记录所有遇到问题

#### 第6-7天：全面切换
- **关闭旧系统**：停止使用freekit.dev链接
- **问题修复**：解决最后的问题
- **正式切换**：宣布迁移完成

### 风险应对计划
#### 风险1：新系统访问问题
- **应对方案**：保留freekit.dev作为备用
- **切换时间**：5分钟内恢复

#### 风险2：手环更新遗漏
- **应对方案**：现场准备备用手环
- **处理流程**：立即写入，5分钟完成

#### 风险3：教师操作不熟练
- **应对方案**：安排技术支持值班
- **支持方式**：电话、微信即时支持

#### 风险4：数据丢失
- **应对方案**：每日数据备份
- **恢复流程**：从备份文件恢复
